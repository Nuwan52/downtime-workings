Recommended model: llama3.1:8b for a standard server (8–16GB RAM). Use mistral:7b as a lighter alternative. For a more powerful machine (32GB+), llama3.1:70b gives analysis quality closer to Claude.

STEP 1 — SERVER

Install Ollama on a server on your factory LAN

Any machine with a decent GPU (or 16GB+ RAM for CPU-only). Ubuntu 22.04 recommended. Run this on the server:

curl -fsSL https://ollama.com/install.sh | sh

STEP 2 — MODEL

Pull the model you want to use

This downloads the model weights to the server (~4–8GB). Only needed once.

ollama pull llama3.1:8b ollama pull mistral:7b # lighter alternative

STEP 3 — NETWORK ACCESS

Allow other devices on the LAN to reach Ollama

By default Ollama only listens on localhost. Set this environment variable so the Phoenix dashboard (on any LAN device) can call it:

OLLAMA_HOST=0.0.0.0:11434 ollama serve

Or add it permanently to /etc/systemd/system/ollama.service as an Environment line. Then open port 11434 in your firewall — but only to the factory LAN subnet, not the internet.

STEP 4 — TEST

Verify from another machine on the LAN

Replace 192.168.1.50 with your server's actual LAN IP:

curl http://192.168.1.50:11434/api/generate \ -d '{"model":"llama3.1:8b","prompt":"hello","stream":false}'

If you get a JSON response back, the Phoenix analyser is ready to connect.

STEP 5 — PHOENIX ANALYSER

Point the analyser at your Ollama server

In the AI analyser tab: select "Local (Ollama)", enter your server IP, pick your model, and run. No other changes needed — the request format is compatible.

API difference — Anthropic vs Ollama

Anthropic (cloud)

POST api.anthropic.com/v1/messages model: claude-sonnet-4-20250514 messages: [{role,content}]

Ollama (local)

POST <server-ip>:11434/api/chat model: llama3.1:8b messages: [{role,content}]

The messages array format is identical. The only changes are the URL, the model name, and no API key header needed.
